## EDU Tabulação — migração e correções (produção)

- Stack do EDU foi migrada para execução sem Docker.
  - `edu-api.service` (NestJS) e `edu-web.service` (Next.js) em systemd, ambos ativos.
  - Nginx configurado para servir EDU em `:8788` com proxy para web (`3000`) e API (`/api` -> `3001`).
- Banco PostgreSQL local configurado com DB `tabulacao` e role `app`.
- Login e rotas básicas do painel EDU validadas em produção (`/login`, `/dashboard`).

## Funcionalidade de alunos/importação

- Criada turma base para testes:
  - Nome: `Clínica A`
  - Código: `CLINICA-A-2026`
  - ID: `4ed0d4b6-557f-45b7-97eb-ae2caeab9477`
- Backend de importação de alunos atualizado para aceitar vínculo de turma por:
  - `classId` direto, ou
  - `classCode`/`turmaCodigo`, ou
  - `className`/`turma`/`classe` (case-insensitive por nome).
- Página `/students` refeita para operação real:
  - criação manual com dropdown de turma,
  - upload Excel/CSV na própria tela,
  - feedback real de sucesso/erro retornado pela API,
  - listagem de alunos com turma resolvida.
- Teste real de importação executado via API com sucesso (HTTP 201, registros criados no banco).
